# Quick Start Guide

Get the pipeline running against your own security tools in six steps.

---

## Prerequisites

```bash
pip install openpyxl requests python-dotenv
```

---

## Step 1 — Configure your environment

```bash
cp .env.example .env
```

Edit `.env` with your API credentials. Required for each connector you want to run:

```bash
# Cloud Security Platform
WIZ_CLIENT_ID=your_client_id
WIZ_CLIENT_SECRET=your_client_secret
WIZ_API_URL=https://api.<your-tenant>.wiz.io/graphql

# Issue Tracker
JIRA_URL=https://your-org.atlassian.net
JIRA_TOKEN=your_api_token
JIRA_PROJECTS=PROJ1,PROJ2
JIRA_SIA_FIELD=customfield_10000   # discover with GET /rest/api/2/field

# Endpoint Management
TANIUM_URL=https://your-tanium-instance
TANIUM_TOKEN=your_api_token

# TLS — if your network uses SSL inspection
CA_BUNDLE=/path/to/corporate-ca-bundle.crt  # omit to use system trust store
```

---

## Step 2 — Convert your Excel SSP to OSCAL

```bash
python scripts/excel_to_oscal.py \
  --input your-ssp.xlsx \
  --output oscal/sspp.json \
  --crosswalk crosswalk/framework-crosswalk.json
```

**What this produces:** An OSCAL JSON file with one evidence slot per tool per control. Every slot starts empty — `api-ready: true` but `last-api-pull: never`. The reconciler ignores empty slots until a connector populates them.

**If your Excel layout differs:** Update `SHEET_NAME`, `HEADER_ROW`, and `COLUMNS` at the top of `excel_to_oscal.py` to match your file.

---

## Step 3 — Run API connectors

Each connector pulls from its tool and writes evidence into the relevant OSCAL slots:

```bash
python scripts/wiz_ingest.py --oscal oscal/sspp.json
python scripts/jira_ingest.py --oscal oscal/sspp.json
python scripts/tanium_ingest.py --oscal oscal/sspp.json
```

**What each one does:**

- `wiz_ingest.py` — pulls open findings from Wiz GraphQL API, maps each finding to NIST control IDs dynamically based on finding characteristics, writes into Wiz slots, archives raw pull to `evidence/wiz/YYYY-MM-DD.json`
- `jira_ingest.py` — runs JQL queries for account requests, change tickets with security impact analysis, incident records, and POA&M items. Writes process control evidence into Jira slots.
- `tanium_ingest.py` — pulls patch compliance by severity SLA and disk encryption status. Writes into Tanium slots. CSV fallback available for on-prem instances without REST API.

---

## Step 4 — Reconcile evidence vs. claims

```bash
# Run reconciler
python scripts/reconcile_oscal.py \
  --oscal oscal/sspp.json \
  --output oscal/sspp.json

# Preview changes without writing
python scripts/reconcile_oscal.py \
  --oscal oscal/sspp.json \
  --dry-run
```

**What this produces:**
- Updated `sspp.json` with reconciler status on every control
- `evidence/reconciliation-report.json` — the structured ISSO brief

---

## Step 5 — Review the output

```bash
cat evidence/reconciliation-report.json
```

Focus on these sections in order:

| Section | What to look for |
|---|---|
| `contradicted_controls` | SSP claims that don't match evidence — act on these first |
| `data_quality_flags` | Tool data issues that look like failures but aren't — process problems, not control failures |
| `partial_controls` | Evidence exists but gaps remain — assess whether compensating controls apply |
| `draft_needed_controls` | No evidence at all — ISSO action required |
| `next_steps` | Prioritized action list generated by the reconciler |

---

## Step 6 — ISSO review and sign-off

The reconciliation report is a brief, not a verdict. The ISSO:

1. Reviews each flagged control with the evidence context provided
2. Applies judgment on compensating controls and approved exceptions
3. Refines auto-generated narratives where needed
4. Escalates true gaps to remediation owners
5. Signs off on the validated output before submission

This is where the GRC agent layer (in development) will pre-populate the brief with a recommendation — so the ISSO validates a conclusion rather than deriving one from scratch.

## Troubleshooting

See [`docs/TROUBLESHOOTING.md`](TROUBLESHOOTING.md) for common issues:

- SSL inspection breaking pip installs and API calls
- Windows Python setup (`py` vs `python`)
- Jira custom field ID discovery
- Wiz tenant field availability variations
- Git credential and branch protection issues
