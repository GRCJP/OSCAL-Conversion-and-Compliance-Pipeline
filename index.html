OSCAL Compliance
Automation Pipeline

A working pipeline that converts a stale Excel SSP into a living OSCAL document — evidence pulled daily from real security tool APIs, reconciled automatically, reviewed by a human ISSO.

NIST SP 800-53 Rev 5 ● OSCAL 1.1.2 ● 19 Tool Connectors ● by-components Architecture
①
What this is
A pipeline that pulls evidence from security tools (Wiz, Jira, Tanium, Splunk) and writes it into an OSCAL System Security Plan automatically, every day. The SSP is always current.
②
The problem it solves
Your SSP says a control is "Implemented." Your tools tell a different story. The reconciler compares SSP claims against real API evidence and surfaces gaps with 5 outcome states — not just pass/fail.
③
Try it — it's interactive
Click a pipeline step to learn what it does. Click a control to inspect its real OSCAL evidence. Hit ▶ Run Reconciler to watch it evaluate every control live from the sample data.
④
The end goal
An AI agent that pre-determines implementation accuracy — reasoning about compensating controls and approved exceptions — so the ISSO reviews a structured brief instead of chasing evidence manually.
